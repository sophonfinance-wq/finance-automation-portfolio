Two honest tiers

The tiers are documented and kept separate — never blended into one inflated headline.

Curated suite 69,911 · default

Hand-written unit and behaviour tests plus parametrized coverage across all ten systems, including a bounded invariant grid on every engine asserting exact identities — sum-preservation, banding, round-trips. Runs with pytest -m "not site_tooling" in minutes and gates every CI build.

Property sweep ~1.26M · opt-in

Exhaustive itertools.product grids asserting sum-preservation, exact integer round-trips, arithmetic identities and determinism across the full integer input domain. Runs on demand with SWEEP=1 pytest -m "not site_tooling" — kept out of the default suite for speed.

A separate 51-test site_tooling suite validates the datasheet generator, schemas, freshness, accessibility, and page budgets. Run it with pytest -m site_tooling; it is deliberately excluded from the 69,911 curated engine total. Every public example runs on fictional data — enforced by a deny-list confidentiality linter that is itself part of the engine test suite. Client information never appears in public code.
